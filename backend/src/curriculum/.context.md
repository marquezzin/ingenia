# App: curriculum

## Propósito
Estrutura de conteúdo pedagógico da plataforma: módulos, aulas, videoaulas, exercícios e casos de teste para correção automática.

## Models
- **`Module`** — Unidade temática da trilha. Campos: `title`, `description`, `sequence_order` (unique), `publication_status`
- **`Lesson`** — Aula dentro de um módulo. Campos: `module` (FK), `title`, `written_content`, `sequence_order`, `publication_status`. Unique: `(module, sequence_order)`
- **`VideoLesson`** — Videoaula 1:1 com Lesson. Campos: `lesson` (O2O), `title`, `video_url`, `duration_seconds` (null)
- **`Exercise`** — Exercício prático vinculado a uma aula. Campos: `lesson` (FK), `title`, `statement`, `support_message` (null), `sequence_order`, `publication_status`. Unique: `(lesson, sequence_order)`
- **`ExerciseTestCase`** — Caso de teste para correção automática. Campos: `exercise` (FK), `name`, `input_data` (null), `expected_output`, `sequence_order`, `is_hidden`. Unique: `(exercise, sequence_order)`

## Enums (`enums.py`)
- **`ContentStatus`**: DRAFT, PUBLISHED, ARCHIVED

## Regras de Negócio
- **BR-006**: Ordenação única de módulos por `sequence_order` (unique constraint)
- **BR-007**: Ordem das aulas é única dentro de cada módulo (unique_together)
- **BR-008**: Aula só pode ser publicada se possuir conteúdo escrito e videoaula associada (validação no service layer)
- **BR-009**: Exercício vinculado a uma aula via FK (not nullable)
- **BR-010**: Exercício só pode ser publicado se possuir ao menos um caso de teste (validação no service layer)

## Endpoints

### Module CRUD (admin only) — `api/v1/modules/`
| Método | URL | Descrição |
|--------|-----|-----------|
| GET | `/api/v1/modules/` | Lista paginada (inclui `description` e `lesson_count`), filtro por `publication_status`, busca por `title`, ordenação por `sequence_order` |
| POST | `/api/v1/modules/` | Criar módulo |
| GET | `/api/v1/modules/:id/` | Detalhe (inclui `lesson_count`) |
| PUT | `/api/v1/modules/:id/` | Atualizar módulo |
| DELETE | `/api/v1/modules/:id/` | Remover módulo |

### Lesson CRUD (admin only) — `api/v1/modules/:moduleId/lessons/`
| Método | URL | Descrição |
|--------|-----|-----------|
| GET | `/api/v1/modules/:moduleId/lessons/` | Lista paginada, filtro por `publication_status` |
| POST | `/api/v1/modules/:moduleId/lessons/` | Criar aula (com VideoLesson inline opcional) |
| GET | `/api/v1/modules/:moduleId/lessons/:id/` | Detalhe (inclui `video` nested e `exercise_count`) |
| PUT | `/api/v1/modules/:moduleId/lessons/:id/` | Atualizar aula + VideoLesson inline |
| DELETE | `/api/v1/modules/:moduleId/lessons/:id/` | Remover aula (cascade deleta vídeo) |

### Exercise CRUD (admin only) — `api/v1/modules/:moduleId/lessons/:lessonId/exercises/`
| Método | URL | Descrição |
|--------|-----|-----------|
| GET | `.../exercises/` | Lista paginada, filtro por `publication_status` |
| POST | `.../exercises/` | Criar exercício |
| GET | `.../exercises/:id/` | Detalhe (inclui `test_cases_count`) |
| PUT | `.../exercises/:id/` | Atualizar exercício |
| DELETE | `.../exercises/:id/` | Remover exercício (cascade deleta test cases) |

### ExerciseTestCase CRUD (admin only) — `.../exercises/:exerciseId/test-cases/`
| Método | URL | Descrição |
|--------|-----|-----------|
| GET | `.../test-cases/` | Lista de test cases |
| POST | `.../test-cases/` | Criar test case |
| GET | `.../test-cases/:id/` | Detalhe do test case |
| PUT | `.../test-cases/:id/` | Atualizar test case |
| DELETE | `.../test-cases/:id/` | Remover test case |

### Student Read (student only) — `api/v1/student/modules/`
| Método | URL | Descrição |
|--------|-----|-----------|
| GET | `/api/v1/student/modules/` | Lista módulos publicados com progresso do aluno |
| GET | `/api/v1/student/modules/:id/` | Detalhe do módulo (com aulas publicadas e progresso) |
| GET | `/api/v1/student/modules/:moduleId/lessons/` | Lista aulas publicadas de um módulo |
| GET | `/api/v1/student/modules/:moduleId/lessons/:id/` | Detalhe da aula (com vídeo, exercícios publicados e progresso) |
| GET | `.../lessons/:lessonId/exercises/` | Lista exercícios publicados (sem test cases) |
| GET | `.../lessons/:lessonId/exercises/:id/` | Detalhe do exercício (sem test cases, com progresso) |

## Services
- **`CreateModuleUseCase`** — Cria módulo com validação de `sequence_order` único
- **`UpdateModuleUseCase`** — Atualiza módulo com validação de `sequence_order` único (excluindo o próprio)
- **`DeleteModuleUseCase`** — Remove módulo por ID
- **`CreateLessonUseCase`** — Cria aula com validação de módulo existente e `sequence_order` único no módulo. Cria VideoLesson inline se fornecido
- **`UpdateLessonUseCase`** — Atualiza aula + gerencia VideoLesson (criar/atualizar/remover conforme payload)
- **`DeleteLessonUseCase`** — Remove aula por ID (cascade deleta vídeo)
- **`CreateExerciseUseCase`** — Cria exercício com validação de aula existente e `sequence_order` único na aula
- **`UpdateExerciseUseCase`** — Atualiza exercício com validação de `sequence_order` único (excluindo o próprio)
- **`DeleteExerciseUseCase`** — Remove exercício por ID (cascade deleta test cases)
- **`CreateExerciseTestCaseUseCase`** — Cria test case com validação de exercício existente e `sequence_order` único no exercício
- **`UpdateExerciseTestCaseUseCase`** — Atualiza test case com validação de `sequence_order` único (excluindo o próprio)
- **`DeleteExerciseTestCaseUseCase`** — Remove test case por ID

## Selectors
- `get_module_by_id(*, id)` — Retorna Module por ID
- `list_modules()` — Lista todos os módulos ordenados por `sequence_order`
- `get_lesson_by_id(*, id)` — Retorna Lesson por ID
- `list_lessons_for_module(*, module_id)` — Lista aulas de um módulo ordenadas por `sequence_order`
- `get_exercise_by_id(*, id)` — Retorna Exercise por ID
- `list_exercises_for_lesson(*, lesson_id)` — Lista exercícios de uma aula ordenados por `sequence_order`
- `get_exercise_test_case_by_id(*, id)` — Retorna ExerciseTestCase por ID
- `list_test_cases_for_exercise(*, exercise_id)` — Lista test cases de um exercício ordenados por `sequence_order`
- `list_published_modules(*, student_profile_id)` — Módulos publicados com Prefetch de progresso do aluno
- `list_published_lessons_for_module(*, module_id, student_profile_id)` — Aulas publicadas de um módulo com progresso
- `list_published_exercises_for_lesson(*, lesson_id, student_profile_id)` — Exercícios publicados de uma aula com progresso

## Serializers
- **`ModuleListSerializer`** — Campos resumidos: id, title, description, sequence_order, publication_status, lesson_count
- **`ModuleDetailSerializer`** — Todos os campos + lesson_count
- **`ModuleCreateUpdateSerializer`** — Campos de escrita para criação/edição
- **`VideoLessonSerializer`** — Leitura: id, title, video_url, duration_seconds
- **`VideoLessonInputSerializer`** — Escrita inline: title, video_url, duration_seconds (optional)
- **`LessonListSerializer`** — Campos resumidos: id, title, sequence_order, publication_status, has_video
- **`LessonDetailSerializer`** — Todos os campos + video nested + exercise_count
- **`LessonCreateUpdateSerializer`** — Campos de escrita + video_lesson nested (optional)
- **`ExerciseListSerializer`** — Campos resumidos: id, title, sequence_order, publication_status, test_cases_count
- **`ExerciseDetailSerializer`** — Todos os campos + test_cases_count
- **`ExerciseCreateUpdateSerializer`** — Campos de escrita: title, statement, support_message, sequence_order, publication_status
- **`ExerciseTestCaseListSerializer`** — Campos resumidos: id, name, sequence_order, is_hidden
- **`ExerciseTestCaseDetailSerializer`** — Todos os campos + timestamps
- **`ExerciseTestCaseCreateUpdateSerializer`** — Campos de escrita: name, input_data, expected_output, sequence_order, is_hidden
- **`StudentModuleListSerializer`** — Campos: id, title, description, sequence_order, lesson_count, progress (nullable)
- **`StudentModuleDetailSerializer`** — Igual ao list + lessons publicadas inline
- **`StudentLessonListSerializer`** — Campos: id, title, sequence_order, has_video, exercise_count, progress
- **`StudentLessonDetailSerializer`** — Todos os campos + video + exercícios publicados + progress
- **`StudentExerciseListSerializer`** — Campos: id, title, sequence_order, progress
- **`StudentExerciseDetailSerializer`** — Campos: id, title, statement, support_message, sequence_order, progress (sem test cases)
