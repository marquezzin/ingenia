# App: submissions

## Propósito
Armazenamento de submissões de código dos alunos e resultados da avaliação automática.

## Models
- **`Submission`** — Submissão de código. Campos: `exercise` (FK → Exercise), `student_profile` (FK → StudentProfile), `source_code`, `evaluation_status`, `score_percentage` (0–100, nullable), `submitted_at`. Indexes: `exercise`, `student_profile`, `(student_profile, exercise, submitted_at)`, `evaluation_status`
- **`SubmissionResult`** — Resultado 1:1 da avaliação. Campos: `submission` (OneToOne), `passed_tests_count`, `failed_tests_count`, `feedback_message`, `result_status`. Index: `result_status`

## Enums (`enums.py`)
- **`SubmissionStatus`**: PENDING, RUNNING, EVALUATED, FAILED
- **`ResultStatus`**: PASSED, FAILED, ERROR

## Regras de Negócio
- **BR-011**: Submissão só pode ser realizada por aluno autenticado para exercício publicado
- **BR-012**: Cada submissão gera exatamente um resultado consolidado (1:1)
- **BR-013**: Feedback automático deve orientar sem expor resposta completa
