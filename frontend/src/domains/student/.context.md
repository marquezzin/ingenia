# Domain: student

## Propósito

Domain frontend para a experiência do aluno na plataforma. Inclui o motor de correção automática client-side via Skulpt, dashboard com trilha de módulos e progresso, listagem e detalhe de módulos, tela de aula com vídeo/conteúdo/exercícios, e as páginas de navegação do aluno.

## Estrutura

### Pages (`pages/`)
- **`DashboardPage.tsx`** — Visão geral com stat cards, card "Continuar de onde parei" e trilha de módulos com progresso visual
- **`modules/ModulesListPage.tsx`** — Lista de todos os módulos com busca por nome, filtro por progresso (SegmentedControl) e grade de cards
- **`modules/ModuleDetailPage.tsx`** — Detalhe de um módulo com descrição, barra de progresso, CTA "Continuar/Iniciar módulo" e lista sequencial de aulas com status
- **`lessons/LessonPage.tsx`** — Tela de consumo de aula com player de vídeo (iframe embed YouTube/Vimeo), conteúdo markdown (react-markdown), lista de exercícios com status, navegação anterior/próxima entre aulas, botão "Concluir aula" para aulas sem exercícios, e auto-mark-started ao acessar
- **`exercises/ExercisePage.tsx`** — Tela de exercício com Monaco Editor (Python), avaliação Skulpt client-side, console de output, painel de resultado com score e feedback pedagógico, submissão ao backend, confirmação antes de limpar código, e botão de dica (support_message)
- **`ProgressPage.tsx`** — Tela de progresso consolidado (`/student/progress`) com stat cards, ring progress, cards de módulos com barra de progresso e contagens, CTA "Continuar estudando"
- **`SubmissionsPage.tsx`** — Tela de histórico geral de submissões (`/student/submissions`) com DataTable filtrável por status (SegmentedControl), paginação, e modal de detalhe do feedback

### UI Components (`ui/`)
- **`ModuleCard.tsx`** — Card de módulo com título, descrição, barra de progresso e badge de status
- **`ContinueStudyCard.tsx`** — Card destaque "Continuar de onde parei" com CTA contextual
- **`DashboardSkeleton.tsx`** — Skeleton de carregamento para o dashboard
- **`LessonItem.tsx`** — Item de aula com ícone de status, título, badge de progresso, e indicadores de vídeo/exercícios
- **`ModulesListSkeleton.tsx`** — Skeleton de carregamento para a lista de módulos
- **`ModuleDetailSkeleton.tsx`** — Skeleton de carregamento para o detalhe do módulo
- **`VideoPlayer.tsx`** — Player de vídeo responsivo (16:9) com iframe embed. Converte URLs YouTube/Vimeo para formato embed
- **`MarkdownContent.tsx`** — Renderiza conteúdo markdown com tipografia estilizada (headings, code blocks, blockquotes, tabelas, listas)
- **`ExerciseListItem.tsx`** — Item de exercício na lista da aula com ícone de status, badge de progresso e contagem de tentativas
- **`LessonPageSkeleton.tsx`** — Skeleton de carregamento para a tela de aula (vídeo + conteúdo + exercícios)
- **`CodeEditor.tsx`** — Wrapper do Monaco Editor para edição de código Python com tema escuro, line numbers e readOnly mode
- **`OutputConsole.tsx`** — Console estilizado monospace (visual terminal) para exibir stdout/stderr do Skulpt
- **`ResultPanel.tsx`** — Painel de resultado com score badge, lista de test cases (PASSED/FAILED/ERROR) com lógica `is_hidden`, e feedback pedagógico destacado
- **`SubmissionHistory.tsx`** — Histórico de tentativas com score, status, data e código expandível por submissão
- **`ExercisePageSkeleton.tsx`** — Skeleton de carregamento para a tela de exercício
- **`ProgressModuleCard.tsx`** — Card individual de progresso por módulo com título, barra de progresso animada, contagem de aulas/exercícios e badge de status
- **`ProgressPageSkeleton.tsx`** — Skeleton de carregamento para a tela de progresso (stat cards + ring + module cards)
- **`SubmissionDetailModal.tsx`** — Modal de detalhe de uma submissão com score, status, feedback pedagógico e código fonte

### Types (`types.ts`)
- **`TestCase`** — Caso de teste recebido da API (input_data, expected_output)
- **`CodeExecutionResult`** — Saída bruta do Skulpt (stdout, stderr, timedOut)
- **`TestCaseResult`** — Resultado por teste (verdict: PASSED/FAILED/ERROR)
- **`EvaluationResult`** — Resultado consolidado (score, feedback, status)
- **`ResultStatus`** — PASSED | FAILED | ERROR
- **`ExecutionState`** — idle | running | complete | error
- **`ProgressStatus`** — NOT_STARTED | IN_PROGRESS | COMPLETED
- **`StudentModule`** — Módulo publicado com progresso do aluno (listagem)
- **`StudentModuleDetail`** — Módulo detalhado com lessons nested e progresso
- **`StudentModuleProgress`** — Progresso em um módulo
- **`StudentLesson`** — Aula publicada com progresso
- **`StudentLessonProgress`** — Progresso em uma aula
- **`ModuleProgressSummary`** — Resumo consolidado de progresso por módulo
- **`StudentVideo`** — Dados de vídeo nested no detalhe da aula
- **`StudentExerciseProgress`** — Progresso em um exercício (status, tentativas)
- **`StudentExerciseListItem`** — Exercício publicado com progresso (listagem)
- **`StudentLessonDetail`** — Detalhe da aula com vídeo, conteúdo, exercícios e progresso
- **`StudentExerciseDetail`** — Detalhe do exercício com statement, support_message, test_cases[] e progresso
- **`SubmissionHistoryItem`** — Submissão do histórico com source_code, score, status e resultado nested
- **`SubmissionHistoryResult`** — Resultado nested de uma submissão (status, contagens, feedback)

### Model (`model.ts`)
- **`getProgressStatusLabel(status)`** — Traduz status para label PT-BR
- **`getLastAccessedModule(modules)`** — Determina o módulo "Continuar de onde parei"
- **`computeOverallProgress(progressList, totalModuleCount)`** — Calcula métricas agregadas
- **`getModuleCompletionPercent(progress)`** — Percentual de conclusão de um módulo

### Services (`services/`)
- **`skulptRunner.ts`** — Executor de baixo nível Skulpt. Configura stdin/stdout, timeout (Sk.execLimit + fallback setTimeout), e delega formatação de erros ao errorHandler.
- **`evaluator.ts`** — Orquestra execução por test case, calcula score e determina status.
- **`feedbackGenerator.ts`** — Gera mensagens pedagógicas (BR-013: sem expor resposta).
- **`errorHandler.ts`** — Formata erros do Skulpt em mensagens amigáveis (pt-BR). Extrai linha de erros de sintaxe/indentação, simplifica runtime errors, e nunca expõe internals.

### API (`api/`)
- **`curriculum.ts`** — GET `/api/v1/student/modules/` (lista), GET `/api/v1/student/modules/:id/` (detalhe), GET `/api/v1/student/modules/:moduleId/lessons/:lessonId/` (detalhe aula), GET `/api/v1/student/modules/:moduleId/lessons/:lessonId/exercises/:exerciseId/` (detalhe exercício com test cases), POST `/api/v1/student/lessons/:id/mark-started/`, POST `/api/v1/student/lessons/:id/mark-completed/`
- **`progress.ts`** — GET `/api/v1/student/progress/` (resumo consolidado) e GET `/api/v1/student/progress/modules/:id/` (detalhe)
- **`submissions.ts`** — POST `/api/v1/student/submissions/` envia resultado avaliado para persistência. GET `/api/v1/student/submissions/?exercise_id=X` lista histórico de tentativas por exercício. GET `/api/v1/student/submissions/` (sem filtro obrigatório) lista todo o histórico com filtro opcional por `evaluation_status`.

### Hooks (`hooks/`)
- **`useStudentModules.ts`** — `useQuery` para lista de módulos publicados + `useStudentModuleDetail(moduleId)` para detalhe com aulas
- **`useStudentProgress.ts`** — `useQuery` para progresso consolidado por módulo
- **`useStudentLesson.ts`** — `useStudentLessonDetail(moduleId, lessonId)` para detalhe da aula, `useMarkLessonStarted()` e `useMarkLessonCompleted()` mutations, `useAutoMarkStarted(lessonId)` para marcar progresso automaticamente ao acessar
- **`useStudentExercise.ts`** — `useStudentExerciseDetail(moduleId, lessonId, exerciseId)` para detalhe do exercício com test cases e progresso
- **`useCodeExecution.ts`** — Hook React para ciclo de execução (idle → running → complete | error).
- **`useSubmission.ts`** — Hook React para envio de resultado ao backend (idle → submitting → success | error).
- **`useExerciseHistory.ts`** — `useQuery` para histórico de submissões de um exercício específico.
- **`useSubmissionHistory.ts`** — `useQuery` para histórico global de submissões do aluno, com filtros opcionais por `evaluation_status`.

## Regras de Negócio
- **BR-012**: Cada submissão gera exatamente um resultado consolidado (1:1)
- **BR-013**: Feedback automático deve orientar sem expor resposta completa
