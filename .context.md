# ingenia — Contexto do Projeto

## O que é
Monorepo full-stack com **Django + DRF** (backend) e **Vite + React + Mantine** (frontend).
Template base para projetos da Synaptha.

## Arquitetura

```
├── backend/src/         Django apps (Python 3.14, uv)
│   ├── config/          Settings, URLs, Celery
│   ├── accounts/        Auth + User model (CORE)
│   ├── ai/              Integração LLM — AIJob, providers (OPCIONAL)
│   └── core/            Utilities: errors, pagination, seed, permissions
├── frontend/src/        Vite + React + TypeScript (pnpm)
│   ├── app/             Bootstrap: rotas, providers, main.tsx
│   ├── shared/          Código compartilhado (HTTP, auth, UI)
│   └── domains/         Um diretório por domínio (espelha apps Django)
│       └── auth/        Login, registro, refresh
├── docker/              compose.yml, Dockerfiles
├── .agent/              Regras e workflows para agentes de IA
└── .issues/             Issues e tracker do projeto
```

## Mapa App Django ↔ Domain Frontend

| Backend App  | Frontend Domain | Descrição                    |
|-------------|----------------|------------------------------|
| `accounts`  | `auth`         | Autenticação JWT, User model |
| `ai`        | *(sem domain)* | Jobs de IA via Celery        |
| `core`      | `shared/`      | Utilitários compartilhados   |

## Convenções Chave

- **Backend**: UseCase pattern (`services/`), selectors (read-only queries), UUID pk, timestamps
- **Frontend**: Domain = 1 app Django. Cada domain tem `api.ts`, `types.ts`, `hooks.ts`, `pages/`, `e2e/`
- **Design System**: Mantine v7 + CSS tokens em `shared/ui/`
- **Testes**: pytest (backend), vitest (frontend unit), Playwright (E2E)

## Para a IA

1. Leia o `.context.md` do app/domain antes de modificar código
2. Consulte `.agent/rules/000-index.md` para regras completas
3. Use workflows em `.agent/workflows/` para tarefas recorrentes
4. Documentação técnica detalhada em `docs/backend/` e `docs/frontend/`
5. Trabalhe com issues em `.issues/` e mantenha o `TRACKER.md` atualizado (ver `006-issues.md`)
